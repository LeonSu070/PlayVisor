# Fast24 Design

## Goal

Build a browser game named Fast24 where the player uses four random numbers from 1 to 10 and the operators `+`, `-`, `*`, and `/` to make 24 as quickly as possible.

## Architecture

Fast24 will be a static single-page web app with no build step. `index.html` owns the screen structure, `styles.css` owns layout and visual treatment, `game.js` owns browser interaction, and `game-core.js` owns pure puzzle and expression logic.

## Screens

The main page shows the title `Fast24` and three controls: `Difficulty`, `Rules`, and `Start Game`. Difficulty opens or reveals a small Easy/Normal/Hard selector. Rules opens a concise rules panel. Start Game moves to the play screen.

The game page shows a timer at the top, four number/result buttons in the center, operator buttons on the right, expression status, completion time when solved, `Give Up`, and `Back to Main Page`.

## Gameplay

The player builds expressions by clicking buttons only. They click a value, an operator, and another value. The game combines those values into a new result button. Each original number can only be used once because combined values replace the two selected inputs. The player repeats this until only one value remains.

The game wins when the only remaining value equals 24. The timer stops and the final completion time is displayed. `Give Up` resets the timer and generates another puzzle. `Back to Main Page` returns to the main screen and clears the active game.

## Puzzle Generation

Each puzzle contains four random integers from 1 to 10. Generated puzzles should be solvable so players are not blocked by impossible sets. Difficulty affects the generated set:

- Easy prefers sets with at least one large or repeated useful value.
- Normal uses any solvable set.
- Hard avoids sets that include 10 when possible.

## Testing

Core logic should be testable in Node without a browser. Tests cover combining two values, preventing invalid division, detecting a win, generating solvable puzzles, and enforcing that each click operation consumes two values into one new value.

# Fast24 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a static click-only Fast24 browser game.

**Architecture:** Use plain HTML, CSS, and JavaScript. Keep arithmetic and puzzle generation in `game-core.js` so it can be tested with Node, and keep DOM state/rendering in `game.js`.

**Tech Stack:** HTML, CSS, browser JavaScript, Node built-in test runner.

---

## File Structure

- `index.html`: app shell and button containers.
- `styles.css`: responsive layout, game controls, modals, and state styling.
- `game-core.js`: pure value-combine, win-check, solvability, and puzzle-generation logic.
- `game.js`: screen navigation, timer, click state, rendering, and user interaction.
- `game-core.test.js`: Node tests for core game behavior.

- [ ] Create `game-core.test.js` with tests for combine, win detection, invalid division, solvability, and puzzle generation.
